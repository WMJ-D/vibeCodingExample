# 中后台管理示例系统

一个基于 Vue 3、Vite、Element Plus 构建的中后台管理示例项目，内置常见管理后台页面、文件预览、地图可视化、配置编辑器、浏览器插件示例、WebRTC P2P 文件互传与局域网视频查看等功能。

## 项目定位

本项目适合作为中后台管理系统原型、前端功能演示项目或业务后台二次开发基础。项目重点覆盖以下能力：

- 标准后台布局、路由、菜单与登录拦截
- 用户、角色、菜单、组织、参数等系统管理页面
- 表单、表格、分页、搜索、新增、编辑、删除等后台高频交互
- 文件本地预览能力
- ECharts 数据看板展示
- AntV L7 地图行政边界可视化
- Chrome 浏览器插件示例
- WebRTC P2P 局域网文件传输
- WebRTC P2P 局域网视频查看
- 外部配置编辑器 iframe 集成

## 核心功能说明

### 1. 后台基础框架

系统采用典型中后台布局，包含登录页、侧边菜单、顶部区域、内容区与路由页面。

- 登录页：提供基础登录入口。
- 路由拦截：未登录访问业务页面时会跳转到登录页。
- 动态标题：根据路由 `meta.title` 自动设置浏览器标题。
- 菜单页面：覆盖首页、系统管理、日志管理、演示功能、配置编辑器等模块。

### 2. 表单与列表功能

项目内置多类后台常见表单与列表交互，可作为 CRUD 页面模板使用。

#### 列表示例

列表示例包含：

- 条件搜索
- 状态筛选
- 数据表格
- 分页切换
- 新增弹窗
- 编辑弹窗
- 删除确认
- 表单校验
- 加载状态
- 模拟接口请求

该页面适合作为普通业务列表页面的基础模板。

#### 用户管理

用户管理页面模拟完整的用户维护流程，包含：

- 用户名、手机号、状态查询
- 用户列表分页
- 用户新增与编辑
- 用户状态启用/禁用
- 重置密码
- 删除用户
- 角色选择
- 所属组织、邮箱、手机号等基础信息维护

#### 系统管理

系统管理模块包含以下页面：

- 用户管理

- 组织管理
- 参数管理

这些页面用于展示后台系统常见的权限、组织与基础配置管理场景。

### 3. 文件预览功能

文件预览页面支持用户在浏览器中选择本地文件并直接预览，不需要上传到服务器。

当前支持：

- 图片预览
- 视频预览
- 音频预览
- PDF 预览
- 文本类文件预览
- JSON、Markdown、HTML、CSS、JavaScript、Vue、Python、Java、C/C++、Go、Rust 等常见文本文件识别
- 文件名、类型、大小展示

实现方式：

- 图片、音视频、PDF 使用 `URL.createObjectURL` 生成本地预览地址。
- 文本文件使用 `FileReader` 读取内容并渲染。
- 页面卸载时释放 Object URL，避免浏览器资源泄漏。

### 4. 地图功能

地图页面基于 AntV L7 与高德底图实现行政边界可视化展示。

主要能力：

- 行政区块填色展示

- 未定边界虚线标识
- 墙体效果图层
- 图例开关控制图层显隐
- 点击行政区块查看名称与编号
- 深色地图风格与可视化面板样式
- 页面卸载时销毁地图实例，避免 WebGL 资源泄漏

地图图层包括：

| 图层 | 描述 |
| --- | --- |
| 行政区划 | 使用半透明填充展示行政区域 |

| 未定边界 | 使用红色虚线表达特殊边界 |
| 墙体效果 | 使用 L7 wall 图层增强视觉层次 |

适用场景：

- 地理数据可视化
- 行政区划展示
- 区域看板
- 大屏地图模块
- 边界数据演示

### 7. P2P 局域网文件互传

局域网互传页面基于 WebRTC DataChannel 实现浏览器之间的 P2P 文件传输。

核心特点：

- 浏览器与浏览器直连
- 文件数据不经过业务服务器
- 支持任意类型文件
- 手动复制 Offer / Answer 完成信令交换
- 支持文件拖拽选择
- 支持多文件待发送队列
- 文件分片传输
- 传输进度展示
- 发送与接收历史记录
- 接收完成后可保存下载
- 支持连接状态展示与重置

传输流程：

1. A 端点击“创建连接”，生成 Offer。
2. A 端复制 Offer 给 B 端。
3. B 端粘贴 Offer，生成 Answer。
4. B 端复制 Answer 给 A 端。
5. A 端导入 Answer，建立 P2P 连接。
6. 连接成功后选择文件并发送。
7. B 端接收文件分片，完成后组装为 Blob 并提供下载。

技术实现要点：

- 使用 `RTCPeerConnection` 创建 P2P 连接。
- 使用 `RTCDataChannel` 传输文件数据。
- 使用 64KB 分片读取文件，避免一次性发送大文件导致内存压力。
- 使用 `bufferedAmount` 与 `bufferedAmountLowThreshold` 做背压控制。
- 通过 JSON 控制消息区分文件元信息、文件结束标记与二进制分片。

注意：当前页面采用手动信令交换，适合演示与局域网场景。如果要用于正式多人在线场景，可以接入 WebSocket 信令服务。

### 8. P2P 局域网视频查看

局域网视频页面基于 WebRTC MediaStream 实现 A 端摄像头视频推送到 B 端浏览器查看。

核心特点：

- A 端打开摄像头
- A 端创建 Offer
- B 端导入 Offer 并生成 Answer
- A 端导入 Answer 后建立连接
- 视频流通过 WebRTC P2P 传输
- B 端只接收远端视频，不采集本地摄像头
- 页面展示本地视频、远端视频与连接状态
- 支持重置连接并释放摄像头资源

适用场景：

- 局域网摄像头预览
- 点对点视频查看
- WebRTC 视频传输学习示例
- 无服务器视频流演示

```bash

```

## 开发说明

### 登录说明

项目包含简单登录拦截逻辑，业务页面会检查本地是否存在 `admin_token`。如果没有 token，会跳转到登录页。

### 地图说明

地图页面依赖在线行政边界 GeoJSON 数据和高德地图底图。使用时需要保证网络可访问相关地图资源。

### WebRTC 说明

P2P 文件互传与视频查看功能依赖浏览器 WebRTC 能力。建议使用新版 Chrome、Edge 等现代浏览器访问。

由于当前采用手动复制 Offer / Answer 的方式交换信令，所以该功能更适合演示、学习与局域网调试。如果要投入生产环境，建议增加：

- WebSocket 信令服务
- 房间管理
- 用户在线状态
- 连接鉴权
- 传输失败重试
- 文件完整性校验

### 插件说明

Chrome 插件示例可作为扩展能力基础。后续可结合后台系统扩展为浏览器侧数据采集、书签管理、页面信息提取、快捷工具入口等能力。

## 浏览器兼容性

推荐环境：

- Chrome 最新版
- Edge 最新版
- 支持 WebRTC、FileReader、Blob、Clipboard API 的现代浏览器
